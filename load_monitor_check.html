ℹ 사용 안내
• OCR 사용법: 로드모니터 화면을 정면에서 밝은 환경으로 촬영
• 인식 값: TOTAL(녹색 큰숫자) / CH1(노랑) / CH2(분홍)
• 색상별 필터 전략: 색상마다 3개 필터를 순서대로 시도 (필터 전 항상 원본 복귀)
  - 녹색: strict→loose→inverse (보통 선명함)
  - 노랑: loose→strict→inverse (어두운 노랑 우선)
  - 분홍: inverse→loose→strict (RGB 단순조건이 카메라 변동에 강함)
• 영역 직접 지정: 자동 OCR이 안되면 ✂ 버튼 → 사진 위에서 TOTAL/CH1/CH2 영역을 드래그
• OCR 결과 확인: 인식 후 값 수정 가능 → 시간대 선택 → ✅ 확인 클릭 시 측정값 입력
• 측정값 셀 수정: 시간대별 측정값 칸을 더블클릭(또는 두번 탭)하면 큰 숫자 키패드 모달이 열림
• 일일 임시저장: 모든 입력값(호기/기종/타발수/측정값)이 자동 임시저장됨 → 하루 동안은 페이지 닫고 다시 와도 자동 복원
• 점검 저장 후: Supabase에 정식 저장 + 측정값/메모/타발수 자동 초기화 (호기·기종은 유지) + 임시저장 삭제
• SBC 기종 룰: SBC로 시작하는 기종명은 앞 4자리만 자동 등록 (예: SBC1234-5678 → SBC1)
• 시간대 자동매칭: 한국 표준시(KST/Asia/Seoul) 기준 ±30분 이내 가장 가까운 시간대에 자동 입력. 사용자 PC 시간대와 무관하게 항상 KST 기준
• 점검자 자동표시: 로그인 정보(ESG_USER)에서 자동 입력
• 호기 마스터: machine_master_tj에서 활성 호기만 로드
• 고객사 마스터: common_code의 code_group=customer
• 기종 마스터: general_model_tj + defect_model 통합 (부분 검색)
• 저장 위치: Supabase load_monitor_check 테이블
